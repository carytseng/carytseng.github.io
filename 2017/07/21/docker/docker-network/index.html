docker-network

Docker-network

  • bridge：桥接方式，宿主机与容器间的网络进行桥接，进行端口映射，共用ip
  • host：去除宿主机与容器间的网络隔离，直接用宿主机的ip
  • overlay：将多个Docker守护程序连接在一起，并使群集服务能够相互通信。
  • None：禁止联网

场景选择

  • 多个容器在同一个主机上通信，使用桥接。默认的桥接会加入同一个网络，生产上建议使用自定义的桥接网络
  • 当不与主机隔离，直接使用主机网络
