Docker

第一章.容器发展之路

单服务器运行单应用的时代，扩展服务容量需要添加服务器，成本高。

使用vmware，即虚拟机技术，会占用额外的cpu、ram和存储。但是虚拟机依赖操作系统，可移植性差，启动慢。

容器与虚拟机最大的区别是不会占用操作系统，而是共享宿主机的资源，而且启动快和便于迁移。

Docker之前有很多虚拟化技术，但由于使用复杂度高得不到广泛应用。

Kubernetes是Docker之上的一个平台，其采用Docker实现其底层容器相关的操作。

第二章.走进Docker

Docker是运行在Linux和Windows上的软件，用于创建、管理和编排容器。

Docker引擎是用于运行和编排容器的基础设施工具.

容器开放计划OCI是一个旨在对容器基础架构中的基础组建进行标准化的管理委员会。

第三章.Docker安装

Docker存储驱动选择，每个Docker容器都有一个存储空间，用于保存层叠的镜像层以及挂载的文件系统，Docker在Linux底层支持几种不同的存储驱动的具体实现，不同的驱动对Docker的稳定和性能有影响.

镜像是包含了os文件系统和应用的对象，等价于未运行的容器。

第五章.Docker引擎

Docker采用模块化设计

Docker daemon实现了Docker API，基于HTTP

对容器的操作由containerd完成，containerd主要负责容器的生命周期管理，它指挥与OCI兼容的容器运行时来创建容器，runc是OCI容器运行时规范的实现，其基于Docker公司开发的Libcontainer与系统内核交互，Libcontainer是为了代替LXC，达到平台无关化。containerd调用runc，确保Docker镜像以OCI bundle的格式给runc。Docker daemon剩下的主要职责有API、镜像管理、身份认证、安全特性、核心网络等。

第八章.应用容器化

docker image build命令会读取Dockerfile,并将应用容器化。使用-t打标签，使用-f指定Dockerfile的路径和名称，-f指定任意路径下的任意名称的Dockerfile

from指定构建的镜像的基础镜像

run用于在镜像中执行命令，每个run创建一个新的镜像层

copy将新的层添加到镜像中

EXPOSE用于记录应用所用的网络端口

ENTRYPOINT用于指定镜像以容器方式启动默认

日志查询

docker logs [option] [container]

Options:

  • –details 显示更多的信息
  • -f, –follow 跟踪实时日志
  • –since string 显示自某个timestamp之后的日志，或相对时间，如42m（即42分钟）
  • –tail string 从日志末尾显示多少行日志， 默认是all
  • -t, –timestamps 显示时间戳
  • –until string 显示自某个timestamp之前的日志，或相对时间，如42m（即42分钟）
